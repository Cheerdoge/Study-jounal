# 触发条件
## 切片
当新添加的元素超出了底层数组的最大容量
## Map
1. 负载因子大于6.5
	* 负载因子可以拿来衡量此时map的性能，该值越高性能越差
	* 相当于平均每个桶有6.5值（最多为8）
2. 新添键值对超出了上限
# 扩容方式
## 切片
在堆上新分配一个2倍的内存（若超过1024则1.25倍），然后将旧的数组元素复制过去，再将切片头指向新数组
* 堆与栈
	* 堆（Heap）
		1. 一般储存内存较大的数据
		2. 数据在其上可以动态增长内存大小
		3. 分配速度相对较慢
		4. 全局可见
	* 栈（Stack）
		1. 后进先出（表现在连续调用函数，最后调用的函数最先结束）
		2. 局部可见（局部变量）
		3. 分配很快
## Map
创建一个原来两倍大小的桶，再在后续每次操作map中一点点搬过去
* 桶
是map储存键值对的基本单位，一个可以装8个
* 哈希表
一种数据结构，就是键值对，提供极快的查找、删除、插入操作（数组和遍历：数组适合遍历，链表适合插入（已知在哪插入））
