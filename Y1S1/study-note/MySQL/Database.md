# 术语
- **数据库:** 数据库是一些关联表的集合。
- **数据表:** 表是数据的矩阵。在一个数据库中的表看起来像一个简单的电子表格。
- **列:** 一列(数据元素) 包含了相同类型的数据, 例如邮政编码的数据。
- **行：**一行（元组，或记录）是一组相关的数据，例如一条用户订阅的数据。
- **冗余**：存储两倍数据，冗余降低了性能，但提高了数据的安全性。
- **主键**：主键是唯一的。一个数据表中只能包含一个主键。你可以使用主键来查询数据。
- **外键：**外键用于关联两个表。
- **复合键**：复合键（组合键）将多个列作为一个索引键，一般用于复合索引。
- **索引：**使用索引可快速访问数据库表中的特定信息。索引是对数据库表中一列或多列的值进行排序的一种结构。类似于书籍的目录。
- **参照完整性:** 参照的完整性要求关系中不允许引用不存在的实体。与实体完整性是关系模型必须满足的完整性约束条件，目的是保证数据的一致性。

# 层次1：数据本身

- **数据**： 对客观事物（如一个员工、一次交易）的记录和描述，是信息的载体。可以是数字、文本、日期、图像等。
    
- **表**： 是数据库中最基本的**组织结构**，用于存储特定类型的数据。类似于Excel中的一个工作表。
    
    - **行**： 也称为 **“记录”** 或 **“元组”**。代表一个具体的实体或项目。例如，`学生表`中的一行就是“张三”这位学生的全部信息。
        
    - **列**： 也称为 **“字段”** 或 **“属性”**。代表实体的某一类特征。例如，`学生表`中的“学号”、“姓名”、“年龄”都是列。
        
- **主键**： 表中**唯一标识**每一行记录的字段（或字段组合）。**不能为空，不能重复**。例如，学生的“学号”可以作为主键。
    
- **外键**： 一个表中的字段，它是**另一个表的主键**。用于建立和加强两个表数据之间的链接，确保**参照完整性**。例如，`订单表`中的“客户ID”字段，就是指向`客户表`主键“ID”的外键。
    

# 层次2：管理系统

- **数据库管理系统**： 这是核心软件，即 **DBMS**。它是用户与数据库之间的接口。
    
    - **功能**： 负责数据的定义、创建、查询、更新、维护以及多用户并发访问控制、安全性管理和备份恢复。
        
    - **常见DBMS**： **MySQL**, **Oracle**, **Microsoft SQL Server**, **PostgreSQL**, **MongoDB** 等。
        

# 层次3：操作语言

- **SQL**： **结构化查询语言**，是与关系型数据库通信的**标准语言**。
    
    - **DDL**： 数据定义语言。用于定义和修改数据库结构。如 `CREATE`（创建表）, `ALTER`（修改表）, `DROP`（删除表）。
        
    - **DML**： 数据操作语言。用于操作数据本身。如 `INSERT`（插入）, `UPDATE`（更新）, `DELETE`（删除）, `SELECT`（查询）。
        
    - **DCL**： 数据控制语言。用于控制访问权限。如 `GRANT`（授权）, `REVOKE`（收回权限）。
        

# 层次4：设计理念

- **数据库模型**： 描述数据、数据关系、数据语义以及一致性约束的概念工具的集合。
    
    - **关系模型**： 当前最主流模型。数据以**二维表**形式组织，表与表之间通过主键/外键建立关系。
        
    - **其他模型**： 层次模型、网状模型、对象模型等。
        
- **范式**： 数据库设计时遵循的规范，目的是减少数据冗余，提高数据一致性和完整性。常见的有一范式、二范式、三范式。
    

# 层次5：高级特性

- **事务**： 一个不可分割的**工作逻辑单元**，由一组SQL操作组成。必须满足 **ACID** 特性：
    
    - **原子性**： 事务中的所有操作要么全部完成，要么全部不完成。
        
    - **一致性**： 事务必须使数据库从一个一致性状态变换到另一个一致性状态。
        
    - **隔离性**： 并发执行的事务之间互不干扰。
        
    - **持久性**： 事务一旦提交，其结果就是永久性的。
        
- **索引**： 类似于书籍的目录，是一种加快数据检索速度的**数据结构**（如B树）。它会占用额外空间，并降低数据插入/更新的速度。
    
- **视图**： 基于一个或多个表的**虚拟表**。它不实际存储数据，而是保存一个查询定义。用于简化复杂查询、隐藏数据细节、提高安全性。1. **关系型数据库**：
    
    - **核心**： 基于关系模型，使用SQL语言。
        
    - **特点**： 强调数据一致性、ACID事务、结构化数据。
        
    - **代表**： MySQL, PostgreSQL, Oracle, SQL Server。
        
    - **适用场景**： 财务系统、ERP、CRM等需要高度一致性和复杂查询的业务。
        
2. **非关系型数据库**：
    
    - **核心**： 不严格遵循关系模型，通常不使用SQL（或使用类SQL）。
        
    - **特点**： 灵活的数据结构（如键值对、文档、图）、易于水平扩展、高并发读写。
        
    - **主要分类**：
        
        - **文档型**： 存储类似JSON的文档。如 **MongoDB**, CouchDB。
            
        - **键值型**： 最简单的存储形式（Key-Value）。如 **Redis**, Memcached。
            
        - **列存储型**： 适合海量数据分析。如 **HBase**, Cassandra。
            
        - **图数据库**： 专注于实体间的关系。如 **Neo4j**。
            
    - **适用场景**： 社交网络、内容管理、物联网、大数据实时分析。
